Zayid's Animation Pack

Complete user manual for scroll-triggered animations

How to Link the Scroll Animation Files

You can use the animation library by linking it via CDN or by downloading the files directly and including them in your project.

1. CDN Method

To use the animation pack via CDN, simply add the following lines to your HTML <head> and right before the closing </body> tag: 

<!-- In <head> -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/zayidAhmed/animation-pack@main/Zayid_Animation_Pack.css" />

<!-- Before </body> -->
<script src="https://cdn.jsdelivr.net/gh/zayidAhmed/animation-pack@main/Zayid_Animation_Pack.js"></script>

2. Local File Method

If you prefer to host the files yourself, download scroll-animation.css and scroll-animation.js and place them in your project directory. Then add the following lines: 

<!-- In <head> -->
<link rel="stylesheet" href="/path/to/scroll-animation.css" />

<!-- Before </body> -->
<script src="/path/to/scroll-animation.js"></script>

Make sure to replace /path/to/ with the actual location of your CSS and JS files.

1. Overview

This scroll animation system allows you to animate elements on your webpage when they enter the viewport. It supports two main categories of animations:

  • Direct animations: Animate individual elements when they themselves become visible.
  • Parent-targeted animations (pt- prefix): Animate child elements collectively when their parent container becomes visible.

This manual guides you through usage, customization, and advanced features.

2. How to Use It

2.1. Including the CSS and JS

You need to include the CSS styles and the JavaScript setup functions:

  • The CSS contains all animation base states and transitions.
  • The JS contains two functions:
    • setupScrollAnimations() â€” observes elements with direct animation classes.
    • setupParentDependentAnimations() â€” observes parents and triggers animations on their child elements with pt- prefixed classes.

2.2. Add Animation Classes to Elements

For direct animations (element-targeted), use classes like:

<h1 class="fade-in">Hello World</h1>
<div class="zoom-in-right">Zoom Right Animation</div>

For parent-targeted animations, add classes with pt- prefix on children, and the animation will trigger when their parent element becomes visible:

<section>
  <h1 class="pt-fade-in">Parent Triggered Fade In</h1>
  <p class="pt-zoom-in-left">This animates when parent enters viewport</p>
</section>

2.3. Custom Threshold Usage

The threshold determines how much of an element needs to be visible before the animation triggers. By default, animations trigger when 10% of the element is visible (threshold: 0.1).

Using data-threshold Attribute

You can customize the threshold for individual elements using the data-threshold attribute:

<!-- Animation triggers when 50% of element is visible -->
<h1 class="fade-in" data-threshold="0.5">50% Threshold</h1>

<!-- Animation triggers when 80% of element is visible -->
<div class="zoom-in-right" data-threshold="0.8">80% Threshold</div>

<!-- Animation triggers when only 5% is visible (early trigger) -->
<p class="blur-in-left" data-threshold="0.05">Early Animation</p>

Parent-Targeted Animations with Custom Threshold

For parent-targeted animations (pt- prefix), set the data-threshold on the parent container:

<!-- All pt- animations trigger when parent is 30% visible -->
<section data-threshold="0.3">
    <h2 class="pt-fade-in-top">Title</h2>
    <p class="pt-fade-in-left">First paragraph</p>
    <p class="pt-fade-in-right">Second paragraph</p>
</section>

Threshold Values Reference

Threshold Value Visibility Required Use Case
0.05 5% Very early animation trigger
0.1 (default) 10% Standard animation timing
0.3 30% Moderate delay
0.5 50% Half-visible trigger
0.8 80% Late animation trigger
1.0 100% Fully visible before animating
Note: The threshold applies to the element itself for direct animations, and to the parent container for pt- animations.

2.4. Customizing Animation Timing

You can customize the duration, delay, and easing of animations using inline CSS styles or CSS custom properties.

Using Inline Styles (Recommended)

Add timing properties directly to your HTML elements:

<!-- Custom delay -->
<h1 class="fade-in" style="transition-delay: 0.3s;">Delayed Animation</h1>

<!-- Custom duration -->
<div class="zoom-in" style="transition-duration: 1.5s;">Slow Animation</div>

<!-- Custom easing -->
<p class="rotate-in" style="transition-timing-function: cubic-bezier(0.68, -0.55, 0.265, 1.55);">Bounce Effect</p>

<!-- Combine all timing properties -->
<div class="blur-in-right" 
     style="transition-delay: 0.2s; 
            transition-duration: 0.8s; 
            transition-timing-function: ease-out;">
    Custom Timing Animation
</div>

Creating Staggered Animations

Use different delays to create staggered animation effects:

<section>
    <h2 class="pt-fade-in" style="transition-delay: 0.1s;">First Element</h2>
    <p class="pt-fade-in" style="transition-delay: 0.2s;">Second Element</p>
    <p class="pt-fade-in" style="transition-delay: 0.3s;">Third Element</p>
    <button class="pt-zoom-in" style="transition-delay: 0.4s;">Last Element</button>
</section>

Animation Timing Properties

CSS Property Description Example Values
transition-delay Time to wait before animation starts 0.2s, 0.5s, 1s
transition-duration How long the animation takes 0.3s, 0.8s, 1.5s
transition-timing-function Animation easing curve ease, ease-in-out, cubic-bezier()

Common Easing Functions

Easing Function Effect CSS Value
Linear Constant speed linear
Ease Slow start, fast middle, slow end ease
Ease In Slow start, fast end ease-in
Ease Out Fast start, slow end ease-out
Ease In Out Slow start and end ease-in-out
Bounce Bouncy effect cubic-bezier(0.68, -0.55, 0.265, 1.55)

Advanced Timing Example

<!-- Hero section with complex staggered animations -->
<section class="hero" data-threshold="0.2">
    <h1 class="pt-fade-in-top" 
        style="transition-delay: 0.1s; 
               transition-duration: 0.8s; 
               transition-timing-function: ease-out;">
        Main Title
    </h1>
    
    <p class="pt-fade-in-left" 
       style="transition-delay: 0.3s; 
              transition-duration: 0.6s;">
        Subtitle with delay
    </p>
    
    <button class="pt-zoom-in" 
            style="transition-delay: 0.5s; 
                   transition-duration: 0.4s; 
                   transition-timing-function: cubic-bezier(0.68, -0.55, 0.265, 1.55);">
        Call to Action Button
    </button>
</section>
Tip: Start with small delays (0.1s - 0.3s between elements) for smooth staggered effects. Longer delays can make the page feel slow to users.

4. When to Use Direct Animations vs Parent-Targeted (pt-) Animations

Scenario Use Animation Type Example Use Case
Animate independent elements on scroll Direct animation (no pt-) Individual text or image appearing on scroll
Animate a group of elements simultaneously when their container is visible Parent-targeted (pt-) Animating multiple headings and paragraphs inside a section as a single unit

5. How It Works Internally

5.1. setupScrollAnimations()

- Takes an array of animation class names (without pt-).

- Finds all elements with these classes.

- Uses an Intersection Observer to watch each element.

- Adds .active class when the element enters viewport (>10% visible).

- Removes .active when the element leaves viewport at bottom.

5.2. setupParentDependentAnimations()

- Takes an array of animation class names with the pt- prefix.

- Finds all elements with these classes.

- Collects their parent elements and sets up Intersection Observer on parents only.

- When the parent enters viewport (>10%), adds .active class to all child elements with the pt- classes.

- When the parent leaves viewport, removes .active from all those children.

7. How to Customize

You need to download the file in order to customize it

7.1. Add Custom Animation Classes

Step 1: Define CSS initial state:

.my-custom-animation {
  opacity: 0;
  transform: translateY(50px);
  transition: opacity 1s ease, transform 1s ease;
}

Step 2: Define CSS active state:

.active.my-custom-animation {
  opacity: 1;
  transform: translateY(0);
}

Step 3: Add your class name to the JS setup functions arrays:

// For direct animation:
setupScrollAnimations([
  'fade-in',
  'my-custom-animation'  // your custom class here
]);

// For parent-targeted animation:
setupParentDependentAnimations([
  'pt-fade-in',
  'pt-my-custom-animation'  // your custom class with pt- prefix
]);

Step 4: Add the class to your elements in HTML:

<h2 class="my-custom-animation">Custom animation triggered on element visibility</h2>

<section>
  <p class="pt-my-custom-animation">Parent-triggered custom animation</p>
</section>

7.2. Adjust Intersection Observer Settings

You can customize the Intersection Observer threshold value inside both setup functions to make animation trigger earlier or later:

const observer = new IntersectionObserver(callback, {
  threshold: 0.1  // triggers when 10% visible
});

Increase for later trigger (e.g., 0.5 for 50%) or decrease for earlier trigger (e.g., 0.05 for 5%).

8. Tips & Best Practices

  • Use transition-delay inline styles on elements to create staggered animation effects.
  • For complex layouts, use parent-targeted animations to control the entire section as a unit.
  • Keep your animation CSS concise and avoid conflicting transform styles to maintain smooth transitions.
  • Test on different screen sizes for responsive animation behavior.

9. Troubleshooting

Issue Possible Cause Solution
Animation never triggers Element or parent not in observer list Confirm class names in JS setup arrays
Animations trigger too early or late Intersection Observer threshold mismatch Adjust threshold value
Animations flicker on scroll Multiple overlapping observers or transition conflicts Simplify CSS transitions, avoid conflicting transforms
Parent animations don't trigger Parent element missing or misidentified Ensure pt- classes are direct children of the observed parent
